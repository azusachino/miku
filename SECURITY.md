# Security policy

## Supported versions

Miku Note is an early-stage project. Security fixes are currently developed against the latest version on the default branch.

## Reporting a vulnerability

Please do not open a public issue for an unpatched vulnerability. Use GitHub's private vulnerability reporting for this repository when available. If it is not available, contact the maintainer
through the email listed in the repository's GitHub profile and include:

- the affected version or commit;
- a concise description of the impact;
- reproduction steps or a minimal proof of concept; and
- any suggested mitigation.

Please avoid sending private vault contents or credentials. We will acknowledge a report when practical, investigate it, and coordinate disclosure after a fix is available.

Miku Note is single-user and login-less today. Treat an exposed instance as trusted-network software and place it behind an appropriate access-control boundary before sharing it beyond that network.
